Overview

As part of front-end development we create UI templates. Obviously we know the order of the components and types of these components. In some cases we may need to load components dynamically. For example, if you have multiple components of same type and want to load one of those components(component factory). There are many cases where we need to load components dynamically. Most common situations are loading component in overlay or if you have set of tabs. In this article I am going to use a tab set.

Let's assume you have two tabs called Action and Actioned. Action tab body is with ActionTabComponent and Actioned tab body is with ActionedTabComponent. When user clicks on tabs we want to load appropriate tab body.

Angular provides different ways to archive to load component dynamically. Most common and easy ways are,

  • Conditionally loading with NgIf or NgSwitch directives
  • Using Angular API with ComponentFactoryResolver and ViewContainerRef.

Through API

Angular has a nice APIs to support dynamic component creation. Actually Angular even support dynamic module on fly. In this article we are focusing more on component. To create a component dynamically use below steps.

  • Create a component which you want to load dynamically. You can use Angular-cli ng g c command for the same.
  • Add newly created component in declarations attribute of NgModule. If you are using Angular-cli it would have done already.
  • Add the component in entryComponents attribute. This step is mandatory to load the component dynamically.
  • Place <ng-container> element in html template.
