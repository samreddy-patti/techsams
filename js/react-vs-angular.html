React vs Angular

Overview

In the market, there are tons of frameworks and libraries for front end application development and this count going up day by day. Though there are many, the most used or trendy ones are React and Angular. Both of these technology goals are the same but using different approaches. Choosing one of these frameworks is always challenging. There are a lot of articles on this topic explaining about up and downsides of each. I am attempting my view to compare both of these in practice. My intention is simple, before choosing anything better we should get hands-on and then decide which suits us. Let's start...

Learning curve

React

React is a JavaScript library developed and maintained by Facebook. It uses plain JavaScript with HTML elements along with custom elements inside the Javascript called JSX. Hence it uses plain JavaScript, if you already know Javascript and HTML it is easy to understand and learn. People who are coming from .Net, Java or other backend technologies are a bit more challenging to understand the concepts. React also uses a concept called state management and we need to understand it properly. React is UI only library - that means you need to learn some other frameworks or libraries to make a proper single page application. For example,

  • Routing - There are lot of routing libraries in market to support routing for React. Refer routing link for list.
  • State container - There are different ways to maintain a state in React. Redux is one for state management.
  • AJAX library - We have freedom to choose any library to make AJAX calls. Most familiar once are - like Fetch API, Rx.js, Axios, JQuery and others.

You have freedom to choose any of these libraries based on need. Which means, when you move to another project and using different combination then you have to learn that as well.

Angular

Angular is a framework developed and maintained by Google. It uses a set of a stack which will be common across the applications. Angular uses Typescript statically type programming style, in-built routing API and Rx.js as AJAX API. Due to statically typed programing style, it is easy to understand the code for the people who are coming from backend technologies like .NET and Java. Angular uses a standard set of a stack. It means - learn Angular and use the same stack in all projects. This will minimise the effort of learning. It doesn't mean that Angular doesn't allow to choose other stacks if you like. For example, we can use Fetch API instead of Rx.js or even we can use React as view in Angular.

Portability

The major challenge with front end components is portability. When we develop the component using one stack, we can use that component only in the same stack. For example, if you develop a button component using React we can use it only in React based application. Hence components tied to technology stack we can't reuse the component in other stacks.

Angular custom elements enable a feature to run Angular components to run in React or other stacks. Angular elements are Angular components packaged as custom elements, a web standard for defining new HTML elements in a framework-agnostic way.

Architecture

React is a library which fit in view of MVC model. As explained in the learning sectionIt it doesn't follow any architecture and we need to choose and design how the application architecture would be. But, React follow the component model and make sure you develop the UI components and integrate each other in a page. We will see how to do this in below sections.

Angular is a framework follows MVC pattern for the application. In reality, most of the front end frameworks follow MV*. where 'C' (controller) is not used. Google described Angular as MVW(Model-View-Whatever), we can replace called View-Model(VM) in Whatever to describe Angular architecture. Refer Angular architecture page to understand better. Angular uses decorator for dependency injection. Angular also follow a component model for UI elements. Each component in Angular is combined with templates, directives, pipes and services. Hands-on

Create project

We can use online editors(Codepen, Plnkr Stackblitz or others) to play around Angular and React features. We can also use our local setup to run these. For local setup, React and Angular need NPM to manage the dependencies. Download Node.js which includes NPM. Once you are done, make sure npm added to classpath. Use below command to verify the same. 

    npm --version

React

Use below command to install the react command line utility in global path. This utility will help to create a react application quickly. 

    npm install -g create-react-app

Once utility is installed use below commands to create the react application. 

    create-react-app react-demo
    cd react-demo/

create-react-app react-demo command will creates a package.json with required React dependencies (react, react-dom, and scripts) and basic folder structure. Use below command to run the application. 

    npm start

Above command will start the http server on port 3000 and opens the browser.

Angular

Angular has CLI which will help to generate the project, folder structure, development, testing, building and deployment. Install Angular CLI with below command. 

    npm install -g @angular/cli

Create Project and folder structure 

    ng new angular-demo
    cd angular-demo

ng new command will creates project, folder structure, installs required dependencies and configures the end to end testing with Protractor. Angular has standard folder structure and configuration. check Angular file-structure and workspace-config for more information.

Use below command to run application 

    ng serve --open

Out of box, Angular provides mocking, proxy, unit testing, functional testing, modularity and so on which will help to reduce lot of setup work.

Conclusion

As mentioned before, React and Angular has its own advantage and disadvantages. Most considerable points are,

React uses dynamically typed programming modal with store concepts. Facebook created React view library with virtual DOM to solve their specific (may be common for all web apps) problem which is displaying bulk data which can change dynamically. JSX modifies a lot of HTML elements or attributes to make it work with React. Each project has free to choose different tech stack to create a framework for an application which increases the learning curve.

Angular uses statically typed programming modal with Typescript. This will help better understanding and maintains for large scale projects. But has lot of boilerplate code or configuration to develop components. Thanks to Angular CLI for minimizing a manual configuration. Angular elements can be used with any other frameworks which help more re-usability. Angular uses all HTML elements and attributes with an extra set of elements to separate Javascript and HTML. Uses a standard set of a stack which minimises the learning curve. Easy to pick for .NET and Java background people.
